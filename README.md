# Задача 2: Брокер сообщений
## Составьте таблицу возможностей различных брокеров сообщений. На основе таблицы сделайте обоснованный выбор решения.

## Решение должно соответствовать следующим требованиям:

*  поддержка кластеризации для обеспечения надёжности,
*  хранение сообщений на диске в процессе доставки,
*  высокая скорость работы,
*  поддержка различных форматов сообщений,
*  разделение прав доступа к различным потокам сообщений,
*  простота эксплуатации.
*  Обоснуйте свой выбор.

# Ответ 
|Критерий|Apache Kafka|RabbitMQ|
|:-----------:|:-----------:|:-----------:|
|Кластеризация|распределённый кластер, репликация|Кластеризация через mirrored queues|
|Хранение|сегментированные лог-файлы|persistent messages, queue durability|
|Скорость работы|Очень высокая (~100K+ сообщ/сек)|Высокая (~50K+ сообщ/сек)
|Форматы сообщений|Бинарный, произвольные данные|Любые форматы, headers, properties|
|Права доступа|ACL через SASL/Kerberos, простые квоты|Управление пользователями,permissions|
|Простота эксплуатации|Средняя, сложная настройка|Высокая, Web UI, простота администрирования|

## Ну основываясь на данных таблицы , подходит RabbitMQ , но т.к мало опыта очень, с Apache Kafka немного больше опыта, то выбрал б Appache Kafka (Опять таки из-за не большого опыта траблшутинга)
